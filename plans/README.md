## Findings considered and rejected (do not re-audit)

- **Concurrent counter "race" in `process-tag-queue`**: not a bug. JS is single-threaded; `summary.x += 1`
  between `await`s is atomic — no lost updates possible.
- **RLS leak on `user_notification_preferences`**: not a bug. Policies are `USING (user_id = auth.uid())`
  with matching `WITH CHECK` (`migrations/20260601027000:42-55`) — correctly row-scoped.
- **`ws` WebSocket DoS via react-email (`pnpm audit` HIGH)**: react-email runs only at template
  deploy/preview time (`packages/email/scripts/deploy-templates.tsx`), never as a network-exposed
  runtime service. Not a production DoS vector. (Optional hygiene: a `pnpm.overrides` bump, not worth a plan.)
- **`.npmrc` token exposure**: `.npmrc:10` uses `${NODE_AUTH_TOKEN}` env expansion — the recommended
  pattern. The `pnpm audit` warning is generic. No secret is committed.
- **LLM responses missing `max_tokens`**: input is length-capped and JSON-schema-validated and prompts
  carry SECURITY notes (`tag-event/handler.ts`); adding `max_tokens` is a one-line hygiene tweak, not a plan.
- **`find_similar_events_by_id` "not exposed to users"**: false — already `GRANT EXECUTE ... TO authenticated, anon`
  (`migrations/20260601029000:84-85`). Reframed as plan 017 (verify/document), not "build it".
- **cron-runner.sh duplicated across 8 dirs (plan 012)**: already solved. The per-dir copies are required
  by the Docker build context (each Railway service builds with `rootDirectory: cron/<dir>`, so `COPY`
  can't reach `../_shared`). A single source lives in `cron/_shared/cron-runner.sh`, `scripts/sync-cron-runner.sh`
  propagates it, and `tests/guards/cron-runner-boundary.test.mjs:22-26` fails the build if any copy drifts.
  Nothing to do — no plan written.
- **`event-review` missing from deploy config (plan 014)**: not a drift. `event-review/` is a shared library
  (`mod.ts` + lib files, **no `index.ts`**). The deploy-cli guard derives deployable functions from dirs that
  contain `index.ts` (`tests/guards/deploy-cli-boundary.test.mjs:36`), so `event-review` is correctly excluded
  and the guard passes. Nothing to do — no plan written.
